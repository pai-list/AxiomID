# AxiomID MCP Server вҖ” Model Context Protocol

## Overview

AxiomID exposes an MCP (Model Context Protocol) server that lets AI agents discover and invoke DID/trust/presence/skill/harvest tools directly. The server follows the [MCP specification](https://spec.modelcontextprotocol.io) using JSON-RPC 2.0 over HTTP.

The server is deployed as a Cloudflare Worker and is the primary machine-to-machine interface for AxiomID's trust and identity layer. This is a **tools-only** server (no resources, no prompts).

## Connection Details

| Property | Value |
|----------|-------|
| **Endpoint** | `POST https://axiomid-backend.amrikyy.workers.dev/mcp` |
| **Protocol** | JSON-RPC 2.0 over HTTP |
| **MCP Version** | `2024-11-05` |
| **Capabilities** | `tools` (no `listChanged` notifications) |
| **Server Name** | `axiomid` |
| **Server Version** | `1.0.0` |
| **Auth** | `X-Shared-Secret` header (see below) |
| **Content-Type** | `application/json` |
| **SSE Introspection** | `GET /mcp` returns plain-text description |

## Authentication

All MCP requests require an `X-Shared-Secret` header with a pre-shared token:

```http
POST /mcp HTTP/1.1
Host: axiomid-backend.amrikyy.workers.dev
Content-Type: application/json
X-Shared-Secret: <your-shared-secret-token>
```

Authentication uses **constant-time comparison** (`crypto.timingSafeEqual`) to prevent timing side-channel attacks. Invalid or missing tokens return error code `-32099` with HTTP 401.

## Protocol Flow

### 1. Initialize

```json
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "initialize",
  "params": {
    "protocolVersion": "2024-11-05",
    "capabilities": {},
    "clientInfo": { "name": "my-agent", "version": "1.0.0" }
  }
}
```

Response:

```json
{
  "jsonrpc": "2.0",
  "id": 1,
  "result": {
    "protocolVersion": "2024-11-05",
    "capabilities": { "tools": { "listChanged": false } },
    "serverInfo": { "name": "axiomid", "version": "1.0.0" }
  }
}
```

### 2. List Tools

```json
{
  "jsonrpc": "2.0",
  "id": 2,
  "method": "tools/list"
}
```

Response contains all 11 available tools (see below).

### 3. Call a Tool

```json
{
  "jsonrpc": "2.0",
  "id": 3,
  "method": "tools/call",
  "params": {
    "name": "did_resolve",
    "arguments": { "did": "did:pi:alice:k3x9m2" }
  }
}
```

## Available Tools

### DID Tools

#### `did_resolve`

Resolve a DID to check for on-chain activity.

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `did` | `string` | Yes | The DID to resolve (e.g. `did:pi:abc123`) |

```bash
curl -X POST https://axiomid-backend.amrikyy.workers.dev/mcp \
  -H "Content-Type: application/json" \
  -H "X-Shared-Secret: $SHARED_SECRET" \
  -d '{
    "jsonrpc": "2.0",
    "id": 1,
    "method": "tools/call",
    "params": { "name": "did_resolve", "arguments": { "did": "did:pi:alice:k3x9m2" } }
  }'
```

Response: `{ "did": "...", "resolved": true, "hasActivity": true/false, "timestamp": 1712345678000 }`

#### `did_create`

Create a new DID for a user.

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `piUsername` | `string` | Yes | Pi Network username |
| `trustLevel` | `number` | No | Initial trust level (0вҖ“1, default 0.1) |

```bash
curl -X POST https://axiomid-backend.amrikyy.workers.dev/mcp \
  -H "Content-Type: application/json" \
  -H "X-Shared-Secret: $SHARED_SECRET" \
  -d '{
    "jsonrpc": "2.0",
    "id": 1,
    "method": "tools/call",
    "params": { "name": "did_create", "arguments": { "piUsername": "alice", "trustLevel": 0.5 } }
  }'
```

Response: `{ "did": "did:axiom:axiomid.app:pi:alice", "piUsername": "alice", "trustScore": 0.5, "created": "2026-07-17T..." }`

### Trust Tools

#### `trust_score`

Get the computed trust score for a DID.

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `did` | `string` | Yes | The DID to check |

Score is a weighted combination: 35% XP (activity), 25% stamps (delegation count), 25% delegation (inverse-square propagated), 15% dialectic (adversarial verification).

#### `trust_delegate`

Delegate trust from one DID to another.

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `delegatorDid` | `string` | Yes | DID granting trust |
| `delegateeDid` | `string` | Yes | DID receiving trust |
| `trustLevel` | `number` | Yes | Trust level (0вҖ“1) |

#### `trust_chain`

Resolve a trust delegation chain between two DIDs (BFS, max 3 hops).

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `sourceDid` | `string` | Yes | Source DID |
| `targetDid` | `string` | Yes | Target DID |
| `callerDid` | `string` | Yes | Caller DID (required for IDOR access control) |

### Presence Tools

#### `presence_heartbeat`

Send an agent heartbeat to mark it online.

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `agentId` | `string` | Yes | Agent identifier |
| `metadata` | `object` | No | Arbitrary metadata key-value pairs |

Heartbeats expire after 10 minutes of inactivity. Uses a Durable Object for state.

#### `presence_status`

Get an agent's current presence status.

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `agentId` | `string` | Yes | Agent identifier |

### Skill Tools

#### `skill_list`

List all unique skills installed in the marketplace. No arguments required.

#### `skill_install`

Install a skill for a user.

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `slug` | `string` | Yes | Skill slug |
| `userDid` | `string` | Yes | User DID |
| `version` | `string` | No | Version to install (default `latest`) |

### Harvest Tools

#### `harvest_query`

Enqueue a research query to the Perplexity-powered harvest system.

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `query` | `string` | Yes | Research query |
| `userDid` | `string` | No | User DID for activity tracking |

Queue processing is concurrency-limited to 15 parallel Perplexity API calls.

#### `harvest_result`

Retrieve a harvest result by job ID.

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `jobId` | `string` | Yes | Harvest job ID returned by `harvest_query` |

## Error Codes

| Code | Meaning |
|------|---------|
| `-32601` | Method not found |
| `-32603` | Internal error (see message for details) |
| `-32099` | Unauthorized MCP client request |

All error responses follow JSON-RPC 2.0:

```json
{
  "jsonrpc": "2.0",
  "id": null,
  "error": { "code": -32099, "message": "Unauthorized MCP client request" }
}
```

HTTP status codes: 200 for successful JSON-RPC responses (including tool errors), 401 for auth failures, 500 for unhandled exceptions.

## Security

- All endpoints require `X-Shared-Secret` header with constant-time comparison
- Trust chain resolution enforces IDOR checks: `callerDid` must match source/target or be admin
- Rate limiting on all routes (per IP + path, tiered by auth)
- CORS restricted to `https://axiomid.app`
- No anonymous state mutations
