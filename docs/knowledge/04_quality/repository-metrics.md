## 2. Metric Definitions

| Metric | Definition |
|--------|-----------|
| **Documentation Coverage** | Feature × 6 doc types (README, Memory, Architecture, API Docs, Tutorial, Examples). Weighted: README=25%, Memory=25%, Architecture=20%, API Docs=15%, Tutorial=10%, Examples=5% |
| **Implementation Coverage** | Formal documentation (Memory + README + Architecture) present for each module. Weighted across src/pages, API routes, lib, components, backend, packages |
| **Developer Experience** | 10 dimensions scored 1-10, weighted equally: Time-to-First-Commit, Quick Start, CONTRIBUTING, Examples, Tutorials, FAQ, Troubleshooting, Onboarding, Contributor Docs, SDK DX |
| **Ecosystem Health** | Average across 8 ecosystem scores: Pi Network, W3C Standards, Cloudflare, Vercel, Next.js, MCP, Browser APIs, AI Agents |
| **Broken Truth Severity** | Catastrophic (Claim exists + Reality fails), Critical (Claim + no implementation), High (code + no tests/tests failing), Medium (outdated docs/wrong config), Low (website/README lag) |
| **Truth Score** | Three counters: Opened (total found), Resolved (fixed), Discovered (newly found). Measured per audit cycle |
