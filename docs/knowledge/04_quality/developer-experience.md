## Dimension 1: Time-to-First-Commit (Score: 7/10)

**What it measures:** How quickly can a new developer clone, run, and make their first commit?

### Steps Required (from README Quick Start)

```
1. git clone
2. cd AxiomID
3. npm install
4. cp .env.example .env.local
5. Fill in: DATABASE_URL, PI_API_KEY, SOVEREIGN_KEY_SALT, auth secrets
6. npx prisma migrate deploy && npx prisma generate
7. npm run dev
```

### Analysis

| Step | Time | Risk | Notes |
|------|------|------|-------|
| Clone | ~30s | Low | Standard GitHub clone |
| Install | ~2-5min | Medium | Dependencies: 48 runtime + 25 dev. `postcss.config.mjs` works out of box |
| Env setup | ~10min | **High** | 4 env vars required but `.env.example` has 20+ placeholders. Developer must know which 4 are essential. Missing: `PI_TOKEN_ENCRYPTION_KEY`, `AUTH_SECRET`, `CRON_SECRET`, `OAUTH_STATE_SECRET`, `SANDBOX_AUTH_BYPASS`, `SANDBOX_DEV_TOKEN`, `NEXT_PUBLIC_SITE_URL`. No indication of which are required vs optional. |
| DB setup | ~5min | **High** | `prisma migrate deploy` requires a running PostgreSQL. No SQLite fallback mentioned (though `AxiomID.Memory/قواعد_البيانات.md:10` mentions local SQLite — this contradicts README). Developer without PostgreSQL gets stuck. No Docker compose file provided. |
| Dev server | ~30s | Medium | `next dev` requires all env vars. Fast once env is configured. |

**Blockers:**
- Requires PostgreSQL with no fallback (Neon or local Postgres)
- 20+ env vars in `.env.example`, unclear which are essential vs optional
- No `docker-compose.yml` for local DB
- `npx prisma migrate deploy` will fail if DB is not running

**Recommendations:**
- Add `docker-compose.yml` with PostgreSQL for local dev
- Categorize env vars: REQUIRED / OPTIONAL / PRODUCTION-ONLY
- Add SQLite fallback for quick start (or mention it clearly)
- Add `npm run setup` script that automates steps 3-6
