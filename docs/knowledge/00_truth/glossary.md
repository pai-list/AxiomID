## A

### Agent
An autonomous AI entity owned by a user, with independent Ed25519 keys, DID, verifiable passport, and delegated spending authority. First-class identity actor in the AxiomID system. See Domain Model → Agent.

### Anchor / Anchoring
Binding a cryptographic hash of a Verifiable Credential to a blockchain (Stellar) for immutable, public verification. `src/lib/stellar-anchoring.ts:9-13`. See Domain Model → Anchor.

### Attestation
A cryptographically signed statement issued by a trusted authority about an identity's attributes. Implemented as W3C Verifiable Credentials. See Domain Model → Attestation.

### AxiomID.Memory
The knowledge base vault (19 files) declared as the Single Source of Truth for the AxiomID project. Located at `/AxiomID.Memory/`. Contains design docs, architecture decisions, and project status.

## C

### Capability
A skill or tool that can be installed on an agent. When installed, the agent gains the ability to perform specific actions. Stored as `Skill` in Prisma.

### Claim (Identity)
An assertion about an identity's attributes, made credible through cryptographic proof or provider attestation. Can be verified on-chain or via Pi API. Has a lifecycle: pending → confirmed → expired.

### Claim (Audit)
A factual statement extracted from documentation that is traced through the Truth Chain during audit. Each claim has evidence (file:line) and a verdict at each layer.

### CodeRabbit
Automated code review tool integrated via GitHub. 8 validated learning patterns documented in AGENTS.md and `AxiomID.Memory/08_Engineering/coderabbit-learning-patterns.md`.

### Credential Pool
Multi-provider authentication credential management system with 9 credential sources and 4 rotation strategies (fill_first, round_robin, random, least_used). Defined in AXIOMID_KNOWLEDGE_BASE.md (old Python agent OS).

## D

### Delegated Trust
A trust relationship where one DID delegates a portion of its trust weight to another. Model exists in Prisma (`DelegatedTrust`) but transitive trust depth not implemented. See Domain Model → Delegated Trust.

### DID (Decentralized Identifier)
A W3C-compliant self-sovereign identifier in format `did:axiom:axiomid.app:pi:{uid}`. Created via `src/lib/did.ts:9-12`. Resolved via `src/lib/did-document.ts:1-98`.

### Dumb Executor
Design principle: Write path is simple append-only. Complex logic lives in the query path. From Cognitive OS architecture.

## E

### Ed25519
Elliptic curve used for sovereign key derivation and VC signing. Implementation in `@axiomid/crypto` package and `src/lib/sovereign-keys.ts`.

### Ephemeral DID
A temporary DID for session-based identity, with expiry and revocation support. Model exists (`EphemeralDid` at `prisma/schema.prisma:499-510`) but no API routes manage it.

### Evidence
Specific file path + line number citations supporting a truth audit finding. Every Broken Truth must cite evidence.

### EVM Wallet
Ethereum Virtual Machine compatible wallet (0x... address). Supported for wallet connect via SIWE (Sign-In with Ethereum).

## H

### Hard Gate
Security principle: Write operations go through UMI router, conflict detection, hash chain verification. From Cognitive OS architecture.

### Hash Chain
SHA-256 chain where each action hash is computed from the previous hash concatenated with the action data. `H_n = SHA-256(H_n-1 || Action_n)`. Implemented in `src/lib/trust-chain.ts:22-46`.

### HMAC State Token
Hash-based Message Authentication Code token for wallet connect, with 5-minute expiry. Prevents replay attacks. `src/lib/oauth-state.ts`.

## P

### Passport
A verifiable digital identity document containing claims, trust score, badges, and cryptographic proof. Published at `/passport/[slug]`. See Domain Model → Passport.

### Pi Browser
The Pi Network's built-in browser. Required for Pi SDK authentication. AxiomID is designed for Pi Browser but accessible from standard browsers (sandbox mode).

### Pi SDK
JavaScript SDK for Pi Network authentication and payments. Loaded dynamically in `src/lib/pi-sdk.ts`. Browser-only — gated behind `typeof window !== 'undefined'`.

### Pi Payment
Blockchain payment on Pi Network. Lifecycle: PENDING → APPROVED → COMPLETED. Integrated with Spend Request for agentic payments.

### Proof
Section of a Verifiable Credential containing the cryptographic signature. Format: `type: Ed25519Signature2020, proofValue: <hex signature>`. See Domain Model → Proof.

### Proof-of-Time-Stake
Strategic concept: Lock tokens for 1 year as economic CAPTCHA. Documented in STRATEGY.md:48-53. No code implementation.

## T

Rank level: Visitor (0 XP) → Citizen (100) → Validator (500) → Sovereign (1000). Determined by XP thresholds in `src/lib/tiers.ts:3-8`.

One of 6 SOUL principles. Append-only hash chain for all actions. Also refers to the actual implementation in `src/lib/trust-chain.ts`. See Domain Model → TrustChain.

Network of trust relationships between DIDs. Referenced in IQRA D3 mesh work (PR #300). Data model includes DelegatedTrust.

### Trust Score
Computed value (0-100) from XP, stamps, tenure, and semantic trust. Formula: `XP×0.5 + Stamps×0.2 + Tenure×0.1 + Semantic×0.2`. `src/lib/trust.ts:23-49`.

### Truth Chain
The 5-layer verification framework used in this audit: Knowledge → Implementation → Deployment → Runtime → Reality.
