## Wallet Flow

1. User taps "Connect Pi Wallet" in `ConnectStep.tsx`
2. `handleConnect()` calls `connectPi()` from wallet context
3. `connectPi()` → `ensurePiInitialized()` → `loadPiSdk()` → `Pi.init()` → `Pi.authenticate()`
4. Pi Browser shows wallet approval dialog (popup on mobile)
5. On success: `{ user, accessToken }` returned
6. Token stored in `localStorage.setItem("pi_access_token", token)`
7. `POST /api/auth/pi` creates/updates user in DB
8. Wallet address stored as `pi:{uid}` in User model

**Missing features:** No wallet disconnect. No wallet address change detection. No multi-wallet support.

## Auth Flow (Full Chain)

```
Pi Browser
  ↓ window.Pi.authenticate(["username", "payments"], callback)
  ↓ 45s timeout
User approves in Pi Browser popup
  ↓ { user: { uid, username, name, stellarAddress }, accessToken }
  ↓
connectPi() returns PiSdkAuthResult
  ↓
wallet context stores user + token
  ↓
POST /api/auth/pi { accessToken, uid, username }
  ↓ Zod validation → Rate limit check → Sandbox check
  ↓ GET https://api.minepi.com/v2/me (Authorization: Bearer <token>)
  ↓ Verify UID match → Extract stellar address → Verify username
  ↓ UPSERT User in Neon PostgreSQL
  ↓ Return { userId, walletAddress, stellarAddress, piUid, piUsername, tier, xp, did, kycStatus, hasAgent }
```
