**Pi Browser risks:**
- **Iframe sandbox:** Pi Browser loads apps inside iframe with `sandbox.minepi.com` referrer. Cross-origin checks may fail (`browser-compatibility.md:84`)
- **HTTPS requirement:** Plain HTTP causes Pi SDK payment failures silently. Portless recommended for local dev (`AGENTS.md:233`)
- **Popup behavior:** Auth popups on mobile are slow — 45s timeout configured (`AGENTS.md:237`)
- **Service Worker:** Not tested — Pi Browser may not support SW, breaking offline page (`browser-compatibility.md:29`)

**Overall: ⚠️ PARTIALLY CERTIFIED** — core flows (auth, payments, share, KYC) confirmed at code level. Real device testing needed for full certification.

## Per-Release Certification Runbook

### Automated Checks (CI)

```bash
# Run browser-specific test suites
npm test -- --testPathPattern="e2e"     # Playwright E2E across Chrome + Edge
npm test -- --testPathPattern="pwa"     # PWA installability tests
npm test -- --testPathPattern="pi-sdk"  # Pi SDK mocking tests
```

### Manual Checks (Pre-Release)

1. Open `https://axiomid.app` in Chrome → verify: load, auth, dashboard, passport
2. Open `https://axiomid.app` in Safari → verify: load, auth, glassmorphism renders
3. Open `https://axiomid.app` in Edge → verify: load, auth (smoke test)
4. Open `https://axiomid.app` in Pi Browser → verify: Pi SDK loads, auth works, share works
5. Test offline: DevTools → Offline → navigate → should show `/offline`
6. Test install: Chrome → Install AxiomID → should launch standalone

### Update Certification

After testing, update the Last Verified date and any changed criteria in this document.

## Cross-Browser Issues Log

| Issue | Browsers | Impact | Workaround | Fixed? |
|-------|----------|--------|------------|--------|
| `navigator.clipboard` requires secure context | Safari, Pi Browser | Share fallback to clipboard may fail | Error caught silently in `sharePassport()` | Not fixable (browser security) |
| `backdrop-blur` disabled for accessibility | Safari | Glassmorphism becomes opaque | CSS fallback via `bg-card` | Not fixable (accessibility) |
| Service Worker offline page | All | `/offline` unreachable on first visit | SW registers on first visit | Known limitation |
| Pi Browser iframe cross-origin | Pi Browser | `window.self !== window.top` may error | Try/catch around frame checks | Handled in code |
| Popup blockers | Chrome, Safari, Edge | Auth popup may be blocked | 45s timeout handles failure | Adequate |
