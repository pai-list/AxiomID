## Update Flow

1. SW checks for updates on every navigation (browser handles this automatically)
2. New SW detected → `install` event fires → `skipWaiting()` called immediately
3. New SW activates → `activate` event deletes old caches → `clients.claim()` takes control
4. On next navigation, new SW serves fresh content

**Cache versioning:** `CACHE = "axiomid-v4"` (`public/sw.js:1`). Increment on deploy to force cache refresh.

**Critical constraint (AGENTS.md:253):** Network-first strategy for HTML documents prevents stale index.html from breaking Next.js chunk hashing after a deploy.
