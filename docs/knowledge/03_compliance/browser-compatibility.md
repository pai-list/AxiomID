## Pi Browser Specifics

Pi Browser is a WebKit-based browser (Safari fork) created by Pi Network Team. Key constraints:

- **UA detection:** `src/lib/pi-sdk.ts:213-214` matches `/Pi Browser|minepi|PiApp/i`
- **Iframe sandbox:** Pi Browser loads apps inside iframe with `sandbox.minepi.com` referrer (`src/lib/pi-sdk.ts:216-221`)
- **HTTPS requirement:** Plain HTTP causes Pi SDK payment failures silently. `portless` recommended for local dev (`AGENTS.md:233`).
- **Popup behavior:** Auth popups on mobile are slow — 45s timeout configured (`AGENTS.md:237`)
- **Pi SDK access:** `window.Pi` only available in Pi Browser. Checked via `checkPiBrowser()`.
- **nativeFeature:** `openShareDialog` and `openConsentDialog` only available inside Pi Browser.

### Fallback Chain (when NOT in Pi Browser)

| Feature | Pi Browser | Outside Pi Browser |
|---------|------------|-------------------|
| Auth | `Pi.authenticate()` | `initiatePiSignIn()` → URL redirect to `https://accounts.pinet.com/oauth/authorize` |
| Share | `pi.nativeFeature.openShareDialog()` | `navigator.share()` → `navigator.clipboard.writeText()` |
| KYC Consent | `pi.nativeFeature.openConsentDialog()` | Returns `null` — caller handles fallback |
| Ads | `Pi.Ads.showAd()` | `NOT_IN_PI_BROWSER` error |
| Payments | `Pi.createPayment()` | `NOT_IN_PI_BROWSER` error |
| Wallet Connect | `Pi.authenticate()` | Sandbox bypass only (dev) |

## Known Browser Issues

| Issue | Browsers Affected | Impact | Workaround |
|-------|-------------------|--------|------------|
| `navigator.clipboard` requires secure context | Safari, Pi Browser | Share fallback to clipboard may fail | Error caught silently in `sharePassport()` |
| `backdrop-blur` disabled for accessibility | Safari (prefers-reduced-motion) | Glassmorphism becomes opaque | CSS fallback exists via `bg-card` |
| Service worker offline page | All (SW not registered) | `/offline` page unreachable | SW registers on first visit |
| Pi Browser iframe cross-origin | Pi Browser | `window.self !== window.top` checks may fail | Try/catch around frame checks |
| Popup blockers | Chrome, Safari, Edge | Auth popup may be blocked | 45s timeout handles failure gracefully |

## Offline Mode

- **Service Worker:** Network-first for navigations, SWR for static assets (`public/sw.js:62-123`)
- **Offline page:** `/offline` served from cache when fetch fails (`public/sw.js:85-86`)
- **Cache name:** `axiomid-v4` (`public/sw.js:1`)
- **Precached assets:** manifest, offline page, icons, logo, banner, favicon (`public/sw.js:3-11`)
- **API bypass:** All `/api/` requests bypass the SW (`public/sw.js:57`)
- **Missing:** No offline-first views for dashboard or passport pages. Offline is limited to static page.
