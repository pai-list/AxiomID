## Agent Identity Flow

```
1. User creates agent via dashboard
   → Agent gets DID: did:axiom:pi:{userPiUid}:agent:{agentName}
   → Agent stored in UserAgent model (Prisma)

2. Agent attestation
   → Code hash computed
   → AST audit performed
   → signAgentAttestationCredential() issues VC
   → VC stored on-chain or in DB

3. Agent operates
   → Heartbeat to PresenceDO every 10 min
   → Can call MCP tools (did_create, trust_delegate, harvest_query)
   → Can sign payloads via /api/agent/sign
   → Can request payments via /api/spend-request

4. Agent verification
   → External agents verify via did_resolve
   → Trust computed via trust_score
   → Delegation chain resolved via trust_chain
```
