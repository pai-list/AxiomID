# ISO/IEC 27001:2022 — Information Security Management

**URL:** <https://www.iso.org/standard/27001>

**Purpose:** ISO 27001 is the international standard for Information Security Management Systems (ISMS). It provides a systematic framework for managing sensitive information, ensuring confidentiality, integrity, and availability through risk assessment, security controls, and continuous improvement.

## Key concepts

- **ISMS (Information Security Management System)** — Systematic approach to managing security
- **Annex A Controls** — 93 controls organised across 4 themes: Organisational, People, Physical, Technological
- **Risk Assessment** — Identifying, analysing, and evaluating information security risks
- **Statement of Applicability** — Documented justification for which controls are in scope
- **PDCA Cycle** — Plan-Do-Check-Act for continuous improvement
- **Supply Chain Security** — Vendor and third-party risk management
- **Incident Management** — Detection, reporting, and response to security events

## How AxiomID uses it

- Security practices align with Annex A Technological controls (access control, cryptography, secure development)
- Cryptographic key management in `packages/crypto/` follows key lifecycle principles
- API route authentication and authorisation patterns implement access control (Annex A 8.1–8.3)
- Environment variable management and secrets handling follow least-privilege principles
- Prisma schema and data models include audit fields (createdAt, updatedAt) for traceability
- Session management and token handling follow secure authentication practices
- Planned: formal ISMS documentation and controls mapping for future certification
