### Data Flow

```
content/docs/*.md → glob import at build time → MDX compiled to React components
                                 ↓
                     Frontmatter parsed (title, order, section)
                                 ↓
                     Sidebar generated from frontmatter order
                     Anchor nav generated from markdown headings
                     Search indexed from frontmatter title + first 100 chars
```

### Search Implementation

```typescript
// No external dependency — simple client-side filter
function searchDocs(query: string, docs: DocIndex[]): DocIndex[] {
  const q = query.toLowerCase();
  return docs.filter(d =>
    d.title.toLowerCase().includes(q) ||
    d.preview.toLowerCase().includes(q)
  );
}
```

Fuse.js only if fuzzy matching required (deferred decision — start simple).

### Content Migration

4 existing sections → 4 `.md` files (intro, stamps, sdk, api-reference) + 2 new files (architecture, kya, passport). Content is copy-pasted from current `DocsContent` component with minimal reformatting.

### Routing

- `/docs/intro` → "Getting Started" page
