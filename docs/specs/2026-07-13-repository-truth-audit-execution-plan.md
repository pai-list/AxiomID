# Repository Truth Audit — Execution Plan

> **For agentic workers:** This plan dispatches 9 agents sequentially (Alpha → Omega). Each agent produces knowledge artifacts in `docs/knowledge/`. Phase dependencies are strict — later agents depend on earlier agents' outputs.

**Goal:** Generate all 39 knowledge artifacts for the AxiomID Engineering Intelligence System, tracing every Claim through the 5-layer Truth Chain.

**Architecture:** Stage 1 is read-only observation. Agents read AxiomID.Memory (20 files) + source code + external standards. Agents write new files to `docs/knowledge/` — no existing files are modified.

**Tech Stack:** No code changes. All artifacts are Markdown (`.md`) or JSON (`.json`).

## Global Constraints

- **Stage 1 is strictly read-only.** No existing files modified. No code changed. Pure observation.
- **Every finding must cite evidence.** File path + line numbers. No opinions without proof.
- **All 5 Truth Chain layers** must be traced for each Claim (Knowledge, Implementation, Deployment, Runtime, Reality).
- **Every document MUST include a metadata header:**

  ```markdown
  Version: X.Y
  Generated: YYYY-MM-DD
  Generated by: [Agent Name]
  Confidence: XX%
  Sources:
  - [file path or glob pattern]
  Last Verified: YYYY-MM-DD
  ```

- **All artifacts go under `docs/knowledge/`** in the hierarchical structure (00-05 directories).
- **English only** for all communication and documentation.
