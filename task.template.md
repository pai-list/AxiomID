# Task Intake — PR / Feature / Change

## 1. Goal
_What exactly needs to happen? One sentence._

## 2. Scope
_Which systems, routes, components, and files are touched._

## 3. Risks
_What could break? Auth? DB schema? CI? UX? Security boundaries?_

## 4. Files
_Literal list of files to create/modify/delete._

## 5. Plan
_Step-by-step execution order. Smallest batches possible._

## 6. Verification
- [ ] `npm run lint`
- [ ] `npm test` (relevant suite)
- [ ] `npm run type-check`
- [ ] `npm run build` (if critical path)
- [ ] Smoke test: _which flows?_
