2. Subsystem Deep-Dive

A. Sovereign Identity (DID)
The system implements a custom did:axiom: method. Identity is established by binding a Pi username to a Stellar public key. The Sovereign Passport is a Verifiable Credential (VC) signed by the AxiomID Issuer, which can be independently verified on-chain via the anchor hash.

B. Trust Score Algorithm
Trust is not a static value but a dynamic probability. It uses a physics-inspired Inverse Square Decay for delegation hops and a Boltzmann Probability distribution to map trust scores to action-weights. The current formula is XP * 0.7 + stamps * 0.3.

C. The Genomic Marketplace
Skills are treated as "Genomes" — Markdown-based manifests containing logic and tests. They are published through a moderation queue and executed in a confined Sovereign Sandbox utilizing an AST scanner to prevent malicious execution.

4. Gap Analysis & Critical Failures

Critical Security Risk (P0): The /api/skills/[slug]/execute route is completely open. Any user can trigger skill execution for any agent if they know the slug. This must be gated by requireAuth() immediately.
Data Integrity Risk (P0): The admin moderation process is not atomic. A failure during the skill update after the moderation record is updated leaves the system in an inconsistent state. Must be wrapped in a prisma.$transaction.
Sovereign Identity Gap: To achieve true "Sovereign" status, the DID documents must include the public keys. Without them, the identity cannot be verified by third-party auditors outside the AxiomID system.
Pi App Store Readiness: The app is 90% ready. The manifest.ts is correct, and all icons exist. The remaining hurdle is the "Unverified" status on the pinet.com domain, which requires final submission to the Pi Developer portal.
