يمثل هذا المستند وثيقة التصميم الأساسية لـ **معمارية خلايا أكسيوم الفركتالية** (Fractal Axiom Cell Architecture). وهو مفهوم يعيد هيكلة قاعدة الكود الضخمة لـ AxiomID (البالغة 1.5M+ LOC) إلى خلايا إدراكية متكاملة ومتناظرة ذاتياً تتكرر عند كل مستوى من مستويات المنظومة.

## 🌀 1. الفلسفة الفركتالية والتناظر الذاتي (Fractal & Self-Similarity Philosophy)

في الطبيعة وفي الكون، تتكرر الأنماط المعقدة بشكل فركتالي (أشرطة الحلزون في الأعاصير تماثل أذرع المجرات، وشبكة العصبونات تماثل الشبكة الكونية). في AxiomID، نطبق هذا المبدأ على هندسة البرمجيات:
* **المستويات الهرمية**:
  1. **الكبسولة (.aix Capsule)**: خلية معرفية فردية تمثل وظيفة أو أداة واحدة.
  2. **الرفيق (Familiar)**: تجمع من الكبسولات يمثل شخصية/وكيل مستقل يدير حالة مستمرة.
  3. **المجلس (Conclave)**: شبكة توبولوجية من الرفاق (Familiars) تتعاون لحل المهام المعقدة.
  4. **المركبة (Vessel)**: الواجهة الرسومية والتطبيقية (لوحة الـ Kanban والـ TUI) الحاضنة للخلايا.
  5. **الشبكة (Axiom Network)**: الاتصال والشبكة السيادية بين خلايا أكسيوم المتعددة.

كل مستوى من هذه المستويات يحتوي على نفس الهيكل الأساسي المكون من **6 أجزاء**، مما يبسط التطوير ويحقق توافقاً تاماً وقابلية لإعادة استخدام الكود (DRY).

## 📐 2. هيكلية خلية أكسيوم (The 6-Component Axiom Cell Spec)

تنقسم خلية أكسيوم إلى ستة مكونات متوازنة تؤدي أدوار الحوكمة، الأمن، التنفيذ، والتخاطب:

```
                          ┌───────────────────────────┐
                          │         Axiom Cell        │
                          │  ┌─────────────────────┐  │
                          │  │  Membrane (Sandbox) │  │
                          │  │   ┌───────────────┐   │  │
                          │  │   │ Nucleus (.aix)│   │  │
                          │  │   └───────┬───────┘   │  │
                          │  └───────────┼───────────┘  │
                          │              ▼              │
Receptors (PSS Ports) <───┼────────── DNA (DID) ────────┼───> Mitochondria (Fiscal Limit)
                          │    (SOUL/Constitution)      │
                          │              ▲              │
                          │              │              │
                          │     Memory (TurboQuant)     │
                          └─────────────────────────────┘
```

### 🧬 2.1 الـ DNA (الهوية والميثاق)
* **المكون**: جواز سفر الوكيل اللامركزي (DID) + الدستور (`CONSTITUTION.md`).
* **الدور**: يحدد الصلاحيات القانونية والتشفيرية، والحدود الأخلاقية والسلوكية للخلية. لا يمكن لأي عملية أن تتم إلا بالتوقيع بمفتاح الـ DID الخاص بالخلية.

### 🛡️ 2.2 الغشاء (Membrane - الأمن الحتمي)
* **المكون**: حاوية معزولة خفيفة تعتمد على بيئة تشغيل WebAssembly (مثل Wasmtime).
* **الدور**: حماية الخلية من الاختراقات الخبيثة وتأمين استدعاءات الأدوات (Tool Execution) عبر جدار أمني محكم (Gargoyle Security Rules).

### ☄️ 2.3 النواة (Nucleus - كبسولة المنطق)
* **المكون**: كبسولات الـ `.aix` البرمجية.
* **الدور**: تحتوي على التعليمات والمهارات الحركية والتفكيرية التي تقوم الخلية بتشغيلها ديناميكياً.

### 🔌 2.4 المستقبلات (Receptors - قنوات التخاطب)
* **المكون**: منافذ بروتوكول PSS (Polar Semantic Space).
* **الدور**: منافذ توبولوجية (7 أبواب) للتخاطب المباشر والثنائي (A2A Communication) وتوزيع الرسائل بصفر تأخير وبصيغة بيانات مضغوطة للغاية.

### ⚡ 2.5 الميتوكوندريا (Mitochondria - ميزانية الطاقة)
* **المكون**: وحدة الرصد والمراقبة المالية x402.
* **الدور**: مراقبة استهلاك الخلية لتوكنز الذكاء الاصطناعي وحدود الصرف المالي لـ USDC لمنع أي استنزاف عشوائي للموارد.

### 🧠 2.6 الذاكرة (Memory - التكميم والذاكرة)
* **المكون**: ذاكرة TurboQuant المكممة.
* **الدور**: تخزين واسترجاع المعلومات بكفاءة سياقية عالية عبر التكميم الدلالي وضغط الـ KV Cache لمنع تشتت النماذج المحلية.

## 🛠️ 3. توزيع لغات البرمجة والأنماط المعمارية (Language & Tech Stack)

للتخلص من تكرار الأكواد والحفاظ على خفة وسرعة التطبيق:
1. **Python**: لغة الذكاء والتوجيه العام وربط النماذج (Orchestration Engine).
2. **Go (Golang)**: لغة الشبكات ومعالجة قنوات التزامن الفائقة لـ JSON-RPC وبوابة الـ API (Mesh/RPC Layer).
3. **Rust**: لغة التشفير والعمليات الرياضية المعقدة (Ed25519, TurboQuant, Constitutional Guard Engine).
4. **WebAssembly (WASM)**: البيئة المعزولة للكبسولات البرمجية لمنع استهلاك مساحة Docker.
5. **Zig**: تحكم منخفض المستوى في البيئات التشغيلية وإدارة ذاكرة دقيقة وخفيفة.
6. **Next.js & React**: لوحات التحكم والبوابات المرئية للـ Vessel.

## 🧪 4. بيئة المحاكاة وتكامل الهجرة (Safe Migration & Simulation Framework)

لا يتم استبدال أي كود حالي في AxiomID بشكل مباشر؛ بل يمر عبر **مرحلة اختبار المحاكاة المزدوجة** (Dual-Agent Simulation):
* **محاكاة السيناريوهات**:
  - يتم تشغيل الخلية الفركتالية الجديدة بالتوازي مع النظام المونوليثي الحالي.
  - يقوم محرك المحاكاة بإرسال نفس الأوامر لكلا المكونين ومقارنة النتائج والتحقق من بقاء التوافق 100%.
  - محاكاة سيناريوهات الـ Deadhand Switch للتأكد من أن الغشاء يقوم بالإيقاف الفوري فوراً عند تخطي الصلاحيات.
  - تشغيل اختبارات أداء الذاكرة والـ memory leak للتأكد من أن التكميم يوفر 60%+ من سياق التوكنز.
