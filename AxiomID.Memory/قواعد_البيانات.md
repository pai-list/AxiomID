# هيكلية وإعدادات قواعد البيانات (Database Architecture)

يستخدم مشروع **AxiomID** نظاماً مرناً لقواعد البيانات يجمع بين بيئة التطوير المحلية سريعة الاستجابة وبيئة الإنتاج السحابية القابلة للتوسع.

---

## 💻 1. قاعدة البيانات المحلية والتطوير (Local Development)
* **المحرك**: SQLite
* **المسار**: `axiomid/prisma/dev.db`
* **رابط الاتصال**: `DATABASE_URL="file:./dev.db"` (يتم تعريفه في ملف `axiomid/.env.local`).
* **مخطط البيانات (Schema)**: تتم إدارته عبر Prisma في الملف `axiomid/prisma/schema.prisma`.

### الأوامر البرمجية الأساسية لإدارة التطوير المحلي:
```bash
# توليد عميل Prisma (Prisma Client)
npx prisma generate

# تحديث وتطبيق التغييرات على قاعدة البيانات المحلية مباشرة دون إنشاء ملفات هجرة
npx prisma db push

# فتح واجهة رسومية لإدارة وتصفح الجداول محلياً
npx prisma studio
```

## ☁️ 2. قاعدة بيانات الإنتاج السحابية (Supabase & ghost.build)
في بيئة الإنتاج والتشغيل السحابي، يتم الاعتماد على قاعدة بيانات **PostgreSQL** الموثوقة والمدارة.

* **الخيار الأساسي: Supabase (Free Tier)** ✅:
  - **المحرك**: PostgreSQL مع ملحق `pgvector` مفعل مسبقاً لدعم تشغيل الذاكرة المتجهية.
  - **الاستضافة**: سحابة Supabase المدارة بالكامل.
  - **متغيرات البيئة**: يتم تكوين الاتصال السحابي عبر `DATABASE_URL` في ملفات التكوين والتشغيل للإنتاج:
    ```env
    DATABASE_URL="postgresql://postgres:<password>@db.<supabase-project-id>.supabase.co:5432/postgres?sslmode=require"
    ```
* **الخيار الاحتياطي: `ghost.build`**:
  - يُبقى كخيار بديل استضافي يدعم نفس البنية والمخطط الهيكلي.

## 🗄️ 3. الجداول الرئيسية في نظام الهوية والعملاء
* **User**: يحتوي على بيانات المستخدمين وعناوين محافظهم الرقمية ورتب التحقق الخاصة بهم (`Visitor`, `Citizen`, `Validator`, `Sovereign`).
* **Integration**: جدول الربط مع الخدمات الخارجية (مثل Twitter, Discord, GitHub, Google).
* **UserAgent**: الجدول الرئيسي الممثل لعميل الذكاء الاصطناعي الخاص بالمستخدم.
