# بروتوكول التحقق والتوثيق (Identity, KYC & KYA Protocols)

يوفر نظام **AxiomID** آليات تحقق هجينة تجمع بين إثبات الهوية البشرية (Proof of Humanity / KYC) والتحقق من سلوك وأمان عملاء الذكاء الاصطناعي المفوضين (Know Your Agent / KYA).

---

## 🪙 1. بروتوكول Pi KYC (التحقق البشري عبر Pi Network)
يتم ربط تطبيق AxiomID بـ Pi Network SDK كبوابة رئيسية للتوثيق البشري المقاوم للهجمات السيبل (Sybil Resistance).

### تدفق التوثيق (Auth Flow):
1. يفتح المستخدم التطبيق داخل **Pi Browser**.
2. يقوم التطبيق بطلب التوثيق عبر `Pi.authenticate`.
3. يحصل التطبيق على رمز وصول مؤقت (`accessToken`) ومعرف المستخدم الفريد (`uid`).
4. يُرسل رمز الوصول إلى خادم AxiomID الخلفي (`/api/auth/pi`) للتحقق منه مع خوادم Pi Network الرسمية (`https://api.minepi.com/v2/me`).
5. بعد نجاح التحقق، يتم إنشاء حساب مستخدم برتبة **Visitor** افتراضياً، ويرتبط بمعرف `pi:uid`.

---

## 🤖 2. بروتوكول KYA (اعرف عميلك الذكي - Know Your Agent)
يهدف KYA إلى بناء مستوى موثوقية وسجل أمان للعملاء المستقلين (Sovereign Agents).
* **إثبات الصلاحيات (Permissions Audit)**: تحديد ومراجعة الصلاحيات الممنوحة للعميل بدقة (مثل صلاحية الشراء، التداول، الوصول لقواعد البيانات).
* **تقييم السلوك (Behavior Score)**: مراقبة وتصنيف سجل أداء العميل وخلوه من السلوكيات الضارة أو الاختراقات الأمنية لمنحه موثوقية أعلى في تنفيذ المهام المالية المعقدة.

---

## 🌐 3. توثيق Google & GitHub والتحقق من العمل البرمجي
لزيادة نقاط الخبرة (XP) والارتقاء برتبة المستخدم، نقوم بدمج بوابات الربط التالية:

### ربط حساب GitHub وتقدير المساهمات (XP Rewards based on Codebase):
* **الهدف**: السماح للمطورين بربط حساباتهم البرمجية والحصول على نقاطReputation بناءً على نشاطهم البرمجي ومساهماتهم الفعلية في العام الماضي.
* **الآلية التقنية**:
  1. يضغط المستخدم على "ربط حساب GitHub".
  2. يتم توجيه المستخدم لصفحة ترخيص OAuth الخاصة بـ GitHub لاستصدار صلاحية قراءة سجل المساهمات العامة (Public Repositories & Commits).
  3. يستقبل الخادم الخلفي رمز الترخيص ويستدعي واجهة GitHub API لجلب:
     - عدد الالتزامات (Commits) التي قام بها المستخدم في العام الماضي.
     - عدد طلبات السحب (Pull Requests) المقبولة.
  4. يتم تحويل هذا النشاط البرمجي إلى نقاط خبرة (XP) وفقاً لجدول مكافآت معتمد (مثلاً: +1 XP لكل 5 Commits موثقة، بحد أقصى 300 XP).
  5. يتم حفظ معرف GitHub البرمجي وحجم المساهمات في جدول `Integration` وتحديث إجمالي نقاط المستخدم وترقيته تلقائياً.

## 💼 4. ربط محافظ العملات الرقمية (Crypto Wallets)
توفير خيارات لربط وإثبات ملكية محافظ العملات الرقمية اللامركزية لترقية المستخدم إلى رتبة **Validator** أو **Sovereign**:
* **دعم شبكات EVM (مثل Ethereum, Base)**: استخدام بروتوكول **Sign-In with Ethereum (SIWE)** للتحقق من ملكية العنوان عبر توقيع رسالة مشفرة بدون كلمة مرور.
* **دعم Solana**: توفير توقيع مشفر متوافق مع محافظ Solana (مثل Phantom).
* **إثبات تاريخ المعاملات (Wallet Activity)**: فحص نشاط المحفظة على البلوكشين (مثل عمر المحفظة، حجم رصيد USDC/Pi، ووجود معاملات ذكية موثوقة) وتدقيقها لمنح مكافآت XP إضافية ترفع موثوقية الحساب.
