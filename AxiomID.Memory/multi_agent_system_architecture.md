يقوم نظام AxiomID على معمارية متطورة للغاية للوكلاء المتعددين (Multi-Agent System) تدعم عزل العمليات، وتنسيق المهام الفرعية، وإضافات الأدوات الخارجية، والربط بالمنصات المختلفة.

---

## 1. بيئات تشغيل الصناديق الآمنة (Sandbox Environment System)

يتم تشغيل الأكواد والعمليات الخطرة أو غير الموثوقة داخل بيئة معزولة بالكامل لضمان سلامة خادم المضيف.

* **تحديد المجلد الأساسي (`base.py:81`):**
  تقوم الدالة `get_sandbox_dir()` بتحديد المجلد الأساسي الخاص بالصندوق المعزول عبر التحقق من المتغير البيئي `TERMINAL_SANDBOX_DIR` أو استخدام المسار الافتراضي.
  
* **تهيئة البيئة وعزل الجلسة (`base.py:311, 351`):**
  تقوم الفئة الأساسية بتهيئة الجلسة (`init_session()`) عبر أخذ لقطة (Snapshot) من المتغيرات البيئية الحالية، والدوال المتاحة، وعزل الأوامر داخل مسار فرعي محدد ومراقب بملفات تتبع (`_cwd_marker`).

* **البيئات المحددة:**
  - **بيئة Docker المحلية (`docker.py:277`):** تشغيل العمليات داخل حاويات Docker صلبة ومقيدة بالموارد الأمنية.
  - **بيئة السحابة الموزعة (`modal.py:164`):** تشغيل الصناديق الآمنة سحابياً عبر منصة Modal مع دعم حفظ الحالة والمزامنة المستمرة.

## 2. نظام تسجيل المنصات (Platform Registry System)

يدير النظام الاتصال بالمنصات الخارجية (Discord, Telegram, Slack, Webhook, إلخ) بطريقة معيارية.

* **السجل العام للمنصات (`platform_registry.py:260`):**
  يوجد كائن عالمي موحد `platform_registry` يدير عملية تسجيل واستدعاء محولات المنصات (Platform Adapters).
  
* **تسجيل المنصات الفرعية (`platform_registry.py:172`):**
  تسجيل المحولات عبر دالة `register(entry)` لتخزين تفاصيل المنصة وطريقة الاتصال في القاموس الداخلي `_entries`.
  
* **إنشاء كائنات المحول (`platform_registry.py:208`):**
  تقوم دالة `create_adapter(name, config)` بالتحقق من الإعدادات وصلاحية التبعيات قبل بناء مثيل للمنصة المطلوبة.
  
* **الواجهة الأساسية للمحول (`base.py`):**
  تحدد فئة `BasePlatformAdapter` الدوال المجردة التي يجب على كل محول تنفيذها مثل:
  - `connect()`: الاتصال بالمنصة.
  - `send()`: إرسال الرسائل الحية.
  - `disconnect()`: قطع الاتصال الآمن.

## 3. نظام استكشاف الإضافات (Plugin Discovery System)

نظام مرن لاستكشاف وتحميل الإضافات ديناميكياً لتوسيع قدرات الوكيل.

* **مصادر استكشاف الإضافات:**
  - **الإضافات المضمنة (`plugins.py:55`):** المجلد الافتراضي المرفق مع الكود الأساسي (`get_bundled_plugins_dir()`).
  - **إضافات المستخدمين (`plugins.py:68`):** الإضافات المسجلة في مجلد المستخدم الخاص (`get_user_plugins_dirs()`).
  - **إضافات المشروع الحالي (`plugins.py:83`):** الإضافات الخاصة بمشروع العميل النشط (`get_project_plugins_dir()`).

* **أنبوب تحميل الإضافات:**
  يتم مسح ملفات `plugin.yaml` وتحميل كود التهيئة في الحزم وتفعيل خطاف التسجيل (`register(ctx)` في `plugins.md:53`) حيث يتم تسجيل الأدوات الفرعية (`ctx.register_tool()`) والخطافات (`ctx.register_hook()`).

## 4. نظام تهيئة الوكلاء (Agent Initialization System)

تهيئة وتكوين الوكيل الذكي بالمتغيرات البيئية وقنوات النقل المفضلة.

* **بدء تشغيل الوكيل (`agent_init.py:139`):**
  تقوم الدالة `init_agent()` ببناء كائن الوكيل وإعداد نموذج التخاطب النشط وربطه بالمزود المناسب.
* **تهيئة واجهة المحادثة وناقل البيانات (`agent_init.py:323, 328`):**
  تحديد وضعية العمل `api_mode = "chat_completions"` وتهيئة وتسخين ذاكرة النقل المؤقتة (`_get_transport()`).
* **بيئات العمل المحلية (`terminal_tool.py:886`):**
  استيراد وربط بيئة العمل المحلية (`LocalEnvironment`) لضمان تشغيل الأدوات ومهام التفاعل داخل الجهاز.

## 5. نظام تفويض الوكلاء الفرعيين (Subagent Delegation System)

يتيح للوكيل الرئيسي تفويض المهام المعقدة لوكلاء فرعيين (Subagents) في سياق معزول وموازي.

* **منع الأدوات الدائرية والمحظورة (`delegate_tool.py:45`):**
  مصفوفة `DELEGATE_BLOCKED_TOOLS` تمنع تكرار تفويض مهام التفويض نفسها للوكلاء الفرعيين لمنع الحلقات اللانهائية المفرغة.
* **مكابح الحماية ورفض الأوامر الضارة (`delegate_tool.py:73`):**
  بروتوكول تفويض آمن `_subagent_auto_deny` يفحص الأوامر الموجهة للوكيل الفرعي ويرفض أي عمليات قد تخرق أمان الخادم تلقائياً.
* **التشغيل الموازي وعمق التفويض (`delegate_tool.py:27`, `agent_init.py:440`):**
  - استخدام `ThreadPoolExecutor` لتشغيل المهام الفرعية في خيوط معالجة موازية حية.
  - تعيين مستوى عمق التفويض (`self._delegate_depth = 0` للوكيل الرئيسي، ويزداد تدريجياً للوكلاء الفرعيين لضبط جدار الاستهلاك والتكرار).
